Spring의 Controller 개발 기술

1. 정통기술
	- controller -> jsp로 이동 시 주로 사용
	- jsp에서 필요한 UI 바로 적용 / 다양한 세션 데이터 등 활용 가능
2. Rest 기법을 바로 반영한 기술
	- 동기 방식에 더 적합
	- controller의 메소드가 바로 응답(다른 html, jsp로 이동하지 않음)
	- 응답 포맷: json, json 배열, 단순 문자열
3. 참고예시
	로그인 된 사람이 메일 -> 카페 -> 블로그 ...
	여러개의 화면 이동 필수, url 변경, 단 상태(session) 유지 지속되어야 함
	jsp 등의 화면 처리가 필요하기도 함
